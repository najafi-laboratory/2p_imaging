Pipeline diagnosis summary

1) Why analysis expects inhibitory neurons
- The plotting/analysis code uses `labels` from `masks.h5` and assumes:
  - -1 = excitatory
  -  0 = unsure
  -  1 = inhibitory
- Inhibitory labels are assigned only when anatomy masks overlap functional masks strongly enough.
- Code path:
  - LabelExcInh.get_mask() uses `ops['meanImg_chan2']` only if `ops['nchannels'] == 2`
  - LabelExcInh.get_label() sets label=1 only if overlap probability > 0.5
- Therefore the analysis can plot inhibitory columns, but only if processing generated any `labels == 1`.

2) Why no inhibitory labels appear
- YH24 comparison session (`YH24LG_CRBL_lobulev_20250701_2afc-533`) is 1-channel:
  - nchannels=1
  - meanImg_chan2 missing
  - so inhibitory labeling is effectively unavailable
- MC11 comparison session is 2-channel:
  - nchannels=2
  - meanImg_chan2 present
  - but labels are only {-1:29, 0:1}, so Cellpose/anatomy overlap never exceeded the inhibitory threshold
- Conclusion: empty inhibitory plots are expected from current processed outputs.

4) Main likely causes
- The analysis expects inhibitory labels because it was written for datasets where anatomy-based labeling exists.
- The chosen YH24 comparison session is 1-channel, so inhibitory labels are not possible there.
- MC11 is 2-channel, but anatomy overlap is still too weak/different to produce label=1 cells.
- The big MC11 ROI loss is primarily a QC-threshold mismatch, especially skew/compact, not just notebook behavior.
- There is also a channel-layout mismatch between cohorts:
  - MC11 processed as nchannels=2, functional_chan=2, align_by_chan=1
  - YH24 comparison session is effectively 1-channel

5) Most actionable next step
- Relax or re-tune MC11 QC thresholds first, especially skew and compact.
- Separately, verify whether MC11 channel 2 is actually the intended anatomy/inhibitory-label channel.
