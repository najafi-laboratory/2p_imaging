# Aligned dF/F and Wheel-Speed Analysis

`aligned_dff_behavior_analysis.py` creates one ordered PDF containing
event-aligned population neural and behavior traces from the validated event
table.

The input event table should come from `event_alignment_validation.py`:

```text
block2_analysis_trials_with_dff_frames.csv
```

That table uses actual Bonsai-performed block-2 event IDs, logger timestamps,
and nearest dF/F frame indices. For the second block, this means the Bonsai
output file is used, not the planned task input CSV. The lower-level
`block2_performed_events_with_dff_frames.csv` is kept for timing validation,
but it has one row per short stimulus presentation and should not be used
directly for trial averaging.

For this session the trial-level table contains all 140 validated motor oddball
events, plus 138 standard-control trial starts. A standard-control trial start
is the first valid standard 0 degree `StimStart` between two consecutive
oddball rows. The count is 138 because one pair of consecutive oddballs has no
standard row between them.

## What It Produces

The code writes one multi-page PDF:

```text
aligned_dff_behavior_all_conditions.pdf
```

Page order:

1. Orientation-aligned events, ordered as 0, 22.5, 45, 67.5, 90, 112.5, 135,
   and 157.5 degrees. With the current files, 0 degrees uses standard trial
   starts between oddballs, and 45/90 degrees use the orientation oddball
   trials.
2. Block 2 sensorimotor oddball conditions, ordered as standard 0 degree,
   omission, halt, orientation 90, and orientation 45.

Each row uses GridSpec with population z-scored dF/F and wheel speed side by
side. The line is the mean and the shaded band is SEM. Legends are placed in
dedicated columns next to each subplot and report the neuron count and trial
count. All grids are off, and top/right spines are hidden.

No PNG files are generated by the current code.

The summary files are:

```text
aligned_dff_behavior_condition_summary.csv
aligned_dff_behavior_skipped_conditions.csv
```

For this session, the validated trial-level table only contains performed block
2. Orientations with no trial-level rows are still shown as empty rows in the
PDF and recorded in the skipped-conditions CSV.

## Notebook Usage

```python
from pathlib import Path
import sys
import importlib
from IPython.display import display

code_dir = Path("/home/ihsan/Desktop/data/Georgia_Tech/Closed_loop/Code/2p")
data_dir = Path("/home/ihsan/Desktop/data/Georgia_Tech/Closed_loop/2p_Data/VW01_20260520_Closed_Loop_test-1556")
output_dir = Path("/home/ihsan/Desktop/data/Georgia_Tech/Closed_loop/Figures/VW01_20260520_Closed_Loop_test-1556/aligned_dff_behavior")
event_table = Path("/home/ihsan/Desktop/data/Georgia_Tech/Closed_loop/Figures/VW01_20260520_Closed_Loop_test-1556/alignment_validation/block2_analysis_trials_with_dff_frames.csv")

if str(code_dir) not in sys.path:
    sys.path.insert(0, str(code_dir))

from ali_analysis import aligned_dff_behavior_analysis
importlib.reload(aligned_dff_behavior_analysis)

result = aligned_dff_behavior_analysis.run_aligned_dff_behavior_analysis(
    data_dir=data_dir,
    output_dir=output_dir,
    event_table_path=event_table,
    pre_sec=2.0,
    post_sec=4.0,
    exclude_duration_outliers=True,
    use_standard_trial_starts=True,
)

display(result.condition_summary)
display(result.skipped_conditions)
print("Combined PDF:", result.pdf_path)
print("Condition summary:", result.condition_summary_path)
print("Skipped conditions:", result.skipped_conditions_path)
```

Add `--use-table-standard-control` if you want to use the standard rows already
stored in the trial-level table rather than rederiving them from the full
performed-event table.

## Notes

The validation step removes duration outliers before building the trial-level
table. The aligned analysis also keeps duration-outlier filtering on by
default as a safeguard. Use `--include-duration-outliers` only when reviewing
the lower-level performed-event table directly.

The current files do not contain validated performed timestamps for every block
in the planned task CSV. True all-block alignment will require Bonsai output and
logger timestamps for those blocks, or another validated timing source.
